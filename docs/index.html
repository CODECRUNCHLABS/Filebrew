1. Install dependencies

You need Node.js 16+ and ffmpeg + ffprobe on your PATH.

The easiest path is via Homebrew:

brew install node ffmpeg

If you don't have Homebrew yet, the install command is one line — copy it from brew.sh.

Use winget (preinstalled on Windows 10/11):

winget install OpenJS.NodeJS
winget install Gyan.FFmpeg

Alternatively, download installers from nodejs.org and ffmpeg.org. For ffmpeg, extract the ZIP and add the bin/ folder to your PATH (Search → "Edit the system environment variables").

Verify both work in a new terminal:

node --version
ffmpeg -version

Debian / Ubuntu:

sudo apt update && sudo apt install -y nodejs npm ffmpeg

Fedora / RHEL:

sudo dnf install -y nodejs ffmpeg

Arch:

sudo pacman -S nodejs npm ffmpeg

2. Get the code & run it

  1. On GitHub, click Fork to make your own copy of the repo.
  2. Clone your fork and start the server: 
    git clone https://github.com/<your-username>/code-crunch-labs-converter
cd localvid-1
npm start
  3. Open http://localhost:3000 in your browser. Pick Video & Audio or Image from the landing page.
  4. To stop the server, press Ctrl+C in the terminal.

No npm install needed. The project has zero npm dependencies — everything uses Node built-ins.

3. Use it

Method A — Browser (drag-and-drop)

  1. Open http://localhost:3000.
  2. Pick Video & Audio Converter or Image Converter.
  3. Image only: choose a resize option at the top (Original / Thumbnail / Small / Medium / Large / Custom).
  4. Drag a file from Finder / Explorer onto the colored format pill (e.g., a .mov onto MP4, or a .png onto WebP).
  5. Or click any pill to open your computer's file picker.
  6. Watch the Processing panel — upload → convert → done. The converted file appears in Library and plays in the right pane when clicked.

Method B — Drop into a watched folder (batch / no-browser)

Drop file hereConverts toOutput lands in
data/videos-to-convert/MP4 (browser-safe H.264 + AAC)data/converted/mp4/
data/audio-to-extract/MP3 (192 kbps)data/converted/mp3/

A polling watcher checks every second. Files must have a stable size for 3 seconds before they're picked up, so partially-copied or actively-recording files are never touched. For non-default formats or image resizing, use the browser UI.

Where files are stored

Everything lives under a single data/ folder (auto-created on first run, or wherever you point DATA_DIR=):

data/
├── conversions.log # append-only log
├── videos-to-convert/ # drop folder → MP4
├── audio-to-extract/ # drop folder → MP3
├── images-to-convert/ # drop folder → JPG
├── uploads/ # browser-upload staging
│ └── <format>/ # mp4/, webp/, mp3/, …
└── converted/ # all output files
    ├── mp4/ webm/ mov/ mkv/ gif/ # video
    ├── mp3/ m4a/ wav/ flac/ # audio
    └── jpg/ png/ webp/ avif/ bmp/ tiff/ # image

Override the location with the DATA_DIR env var:

# macOS / Linux
DATA_DIR=/Volumes/External/converter npm start

# Windows (PowerShell)
$env:DATA_DIR="D:\converter"; npm start

Resized images get a -<size>px suffix (e.g. photo-800px.jpg), so different sizes of the same source can coexist.

How it works

A ~600-line single-file Node server with zero npm dependencies. The browser hits it via plain HTTP; conversions are spawned ffmpeg processes.

Zero dependencies

Just Node built-ins (http, fs, path, child_process) — no npm install needed, ever.

Polling folder watcher

1 Hz poll instead of fs.watch — boring and works identically on macOS, Windows, and Linux.

3-second size stability

A file is only touched after its size has been unchanged for 3 seconds — partially-copied or still-recording files are never read.

H.264 fast path

H.264 sources remux to MP4/MOV (-c copy) instead of transcoding — finishes in seconds even on multi-GB files.

Range-request streaming

The browser's <video> tag seeks into multi-GB files without downloading them — server speaks HTTP 206 Partial Content.

Runtime encoder detection

On startup, the server probes ffmpeg's available encoders and hides format pills it can't actually produce. No silent failures.

Limitations

  • WebM and AVIF encoding are slow. Use MP4 / WebP / JPG if speed matters more than file size.
  • HEVC video sources require transcoding (not remuxing) to MP4 because Chrome and Firefox don't decode HEVC. Expect minutes per minute of source.
  • One conversion at a time. Jobs run serially to avoid CPU thrashing. Multiple uploads queue up and process in order.
  • No authentication. The server binds to localhost by default. If you change PORT to expose it on a LAN, anyone on the network can upload and convert.

Frequently asked questions

Is Filebrew free?

Yes. It is 100% free and open-source under the MIT license. There are no quotas, watermarks, paywalls, or subscription fees. Use it for personal or commercial work without obligation.

Does it upload my files to the cloud?

No. Everything runs locally on your machine. The browser interface talks to a Node.js server on localhost. Your files never leave your computer.

What operating systems does it support?

macOS, Windows, and Linux. It requires Node.js 16 or later and ffmpeg + ffprobe installed on your system. See the Install section for one-line install commands per OS.

What video and audio formats can it convert?

Video output: MP4, WebM, MOV, MKV, animated GIF. Audio output: MP3, M4A, WAV, FLAC. Input accepts essentially everything ffmpeg can read — MP4, MOV, WebM, MKV, AVI, M4V, FLV, WMV, 3GP, TS, MP3, M4A, WAV, AAC, FLAC, OGG, OPUS, WMA, and more.

What image formats can it convert?

Image output: JPG, PNG, WebP, AVIF, BMP, TIFF. Input: JPG, JPEG, PNG, WebP, AVIF, GIF, BMP, TIFF, HEIC, HEIF.

Can it resize images automatically?

Yes. The image converter has preset sizes (Thumbnail 200 px, Small 480 px, Medium 800 px, Large 1920 px) plus a custom pixel size option. Aspect ratio is preserved automatically — the longest edge becomes the requested size.

How do I install it?

Install Node.js and ffmpeg, clone the repo, run npm start, open http://localhost:3000. Per-OS install commands are in the Install section above.

Can I run it on a Raspberry Pi or home server?

Yes. Anywhere Node.js 16+ and ffmpeg run, the converter runs. Set PORT and bind the server to your LAN to use it from your phone or another laptop.

Does it convert MOV from iPhone screen recordings to MP4 for the web?

Yes. iPhone screen recordings are typically HEVC inside a MOV container. The converter detects HEVC and transcodes to H.264 + AAC in an MP4 container so the result plays in Chrome, Firefox, Safari, and Edge.

Is it really free with no catch?

Yes. MIT license, source code on GitHub. Fork it, modify it, ship it commercially — no obligation back to the project.
